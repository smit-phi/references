Introduction to APIs

What is an API?

  • An API is a contract that lets one software system request data or actions from another.
  • Clients send requests; servers process them and return structured responses.
  • Well-designed APIs separate frontend concerns from backend business logic.
  • Teams use APIs to integrate web apps, mobile apps, internal services, and partner systems.

🧭

Good REST habits
  • Nouns in URLs, not verbs.
  • Predictable naming and versioning.
  • Consistent error responses.
Where teams slip
  • Mixing business logic into route handlers.
  • Using POST for everything.
  • Returning 200 for errors.

Status codes and JSON

Status code quick map
  • 200 OK — successful read/update.
  • 201 Created — resource created.
  • 204 No Content — delete success, no body needed.
  • 400/422 — bad request or validation failure.
  • 401/403 — auth failure or forbidden action.
  • 404/500 — missing resource or server error.
JSON basics

    "id": 42,

JSON carries objects, arrays, strings, numbers, booleans, and null. FastAPI automatically parses request JSON into Python-friendly data structures.
Why FastAPI

A modern Python API framework

FastAPI is built around standard Python type hints, automatic OpenAPI documentation, validation through Pydantic, and async-friendly request handling.

Speaker note: This is the transition slide. Reinforce that FastAPI feels productive because editors, docs, validation, and runtime behavior all align around the same types.

What is FastAPI?

  • A high-performance Python web framework for building APIs.
  • Built on Starlette for web features and Pydantic for data validation.
  • Uses Python type annotations as the source of truth for request parsing and docs.
  • Ships with interactive docs through Swagger UI and ReDoc.
Official workflow cues
  • Examples are designed to be copied and run directly in local files.
  • FastAPI encourages virtual environments and direct local execution.
  • Development mode can be started with fastapi dev; production guidance recommends fastapi run.

Performance and async support

Why async matters
  • Async is valuable when a request spends time waiting on I/O, such as databases, HTTP services, or file operations.
  • While one request awaits I/O, the server can keep serving other requests.
  • This improves throughput for many backend workloads without adding threads everywhere.
FastAPI foundation
  • FastAPI documents async/await as a core capability for path operation functions.
  • It inherits high-performance ASGI behavior from Starlette.
  • Teams can mix sync and async endpoints, then optimize hotspots as needed.

FastAPI basics

Routes, parameters, bodies, and responses

This is the part where trainees move from “server is running” to “I can model real application endpoints.”

Creating routes

from fastapi import FastAPI
app = FastAPI()

@app.get("/students")
def list_students():
    return [{"id": 1, "name": "Aisha"}]

@app.post("/students")
def create_student():
    return {"message": "created"}
  • Decorators such as @app.get and @app.post bind URL paths to Python functions.
  • The function return value is serialized to JSON automatically.
  • Use clear route names and keep handlers thin.

Path parameters and query parameters

@app.get("/students/{student_id}")
def get_student(student_id: int, active: bool = True):
    return {
        "student_id": student_id,
        "active": active
    }
How it works
  • student_id is parsed from the URL path and validated as an integer.
  • active is a query parameter with a default value.
  • Type hints drive parsing, validation, editor support, and docs.
Common mistake: forgetting that query parameters arrive as strings at the HTTP level even though FastAPI converts them to typed Python values for you.

Request body and response models

from pydantic import BaseModel

class StudentIn(BaseModel):
    name: str
    age: int

class StudentOut(StudentIn):
    id: int

@app.post("/students", response_model=StudentOut)
def create_student(student: StudentIn):
    return {"id": 101, **student.model_dump()}
  • Body models provide structure and validation for incoming JSON.
  • response_model shapes outgoing data and helps prevent over-sharing internal fields.
  • Swagger UI shows both request and response schema examples automatically.

Student Management CRUD example

students = []

@app.get("/students")
def list_students():
    return students

@app.post("/students", response_model=StudentOut)
def add_student(student: StudentIn):
    item = {"id": len(students) + 1, **student.model_dump()}
    students.append(item)
    return item

@app.delete("/students/{student_id}", status_code=204)
def delete_student(student_id: int):
    global students
    students = [s for s in students if s["id"] != student_id]
Speaker note: This is intentionally in-memory and imperfect. Use it to teach behavior first, then later replace the list with database persistence.

Pydantic models

Validation as part of the API contract

In FastAPI, models are not just data containers; they are contracts that define, validate, and document your API.

Data validation and request schemas

from pydantic import BaseModel, EmailStr, Field

class StudentCreate(BaseModel):
    name: str = Field(min_length=2, max_length=50)
    email: EmailStr
    age: int = Field(ge=16, le=80)
    course: str
  • Use type-specific fields such as EmailStr for richer validation.
  • Field() adds bounds, defaults, and metadata.
  • Invalid requests produce structured validation errors automatically.

Response schemas, optional fields, nested models

Response schema
Expose only what the client needs.
class StudentOut(BaseModel):
    id: int
    name: str
    email: EmailStr
Optional field
Useful for partial data or future enrichment.
phone: str | None = None

Great for domain objects.

Async programming

Concurrency without blocking the server

Async endpoints are most useful when your application waits on external systems such as a database, storage, or another API.

async vs sync and the await keyword

@app.get("/reports-sync")
def get_report_sync():
    data = slow_db_call()
    return {"data": data}

@app.get("/reports-async")
async def get_report_async():
    data = await slow_async_db_call()
    return {"data": data}
  • def defines a synchronous endpoint.
  • async def defines an asynchronous endpoint.
  • await pauses the coroutine while other requests keep moving.
  • Use async only when the called library or dependency supports it cleanly.

FastAPI async endpoint workflow

Request
→
Route handler
→
await DB / API
When async helps
  • High-latency I/O.
  • Many concurrent users.
  • Non-blocking integrations.
When not to force it
  • CPU-heavy processing.
  • Libraries that are sync-only.
  • Codebases where complexity outweighs gain.

Dependency injection

Composable dependencies with Depends()

FastAPI’s dependency system can validate request-driven inputs, manage auth, and provide reusable resources like database sessions.

Reusable dependencies

from fastapi import Depends

def common_params(limit: int = 10, skip: int = 0):
    return {"limit": limit, "skip": skip}

@app.get("/students")
def list_students(params=Depends(common_params)):
    return params
  • Dependencies avoid repeating common setup and parameter logic.
  • They can depend on other dependencies, forming a graph.
  • FastAPI includes dependency parameters in validation and documentation automatically.

Authentication and database dependencies

def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

@app.get("/employees")
def list_employees(db = Depends(get_db)):
    return db.query(Employee).all()
  • A database session dependency centralizes setup and cleanup.
  • The same pattern works for current-user resolution or tenant context.
  • Dependencies keep route functions focused on business behavior.

Database integration

From in-memory lists to persistence

Persistent storage is where APIs become useful to real applications. FastAPI works well with SQLAlchemy and multiple relational databases.

SQLAlchemy integration and ORM basics

# main.py

from fastapi import FastAPI, Depends
from sqlalchemy import create_engine, Column, Integer, String
from sqlalchemy.orm import declarative_base, sessionmaker, Session

# ---------------- DATABASE ----------------

DATABASE_URL = "sqlite:///./training.db"

engine = create_engine(
    DATABASE_URL,
    connect_args={"check_same_thread": False}
)

SessionLocal = sessionmaker(bind=engine)

Base = declarative_base()

# ---------------- MODEL ----------------

class Employee(Base):
    __tablename__ = "employees"

    id = Column(Integer, primary_key=True, index=True)
    name = Column(String, nullable=False)
    department = Column(String, nullable=False)

# Create tables
Base.metadata.create_all(bind=engine)
  • SQLAlchemy maps Python classes to database tables.
  • Common starter options: SQLite for training, PostgreSQL or MySQL for production-grade systems.
  • ORM models are usually separate from API schemas.
  • pip install fastapi uvicorn sqlalchemy psycopg2-binary

Connect and perform CRUD operations

# ---------------- DB DEPENDENCY ----------------

def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()
# ---------------- FASTAPI ----------------

app = FastAPI()

# ---------------- ENDPOINTS ----------------

@app.get("/")
def home():
    return {"message": "Employee API Running"}

# Create Employee
@app.post("/employees")
def create_employee(
    name: str,
    department: str,
    db: Session = Depends(get_db)
):
    employee = Employee(
        name=name,
        department=department
    )

    db.add(employee)
    db.commit()
    db.refresh(employee)

    return employee

# Get All Employees
@app.get("/employees")
def get_employees(db: Session = Depends(get_db)):
    employees = db.query(Employee).all()
    return employees

Authentication & security

Protect routes and identities

Security is not an optional add-on. FastAPI includes reusable security components and OpenAPI integration for OAuth2-style flows.

OAuth2, JWT, password hashing, protected routes

pip install fastapi uvicorn bcrypt python-jose python-multipart
import bcrypt

from fastapi import FastAPI, Depends, HTTPException
from fastapi.security import OAuth2PasswordBearer, OAuth2PasswordRequestForm

from jose import jwt

app = FastAPI()

SECRET_KEY = "secret"

oauth2_scheme = OAuth2PasswordBearer(tokenUrl="login")

# -----------------------------
# Password Hashing
# -----------------------------

# Hash password once
hashed_password = bcrypt.hashpw(
    b"1234",
    bcrypt.gensalt()
)

fake_user = {
    "username": "admin",
    "hashed_password": hashed_password
}

# -----------------------------
# Login
# -----------------------------

@app.post("/login")
def login(form_data: OAuth2PasswordRequestForm = Depends()):

    # Check username
    if form_data.username != fake_user["username"]:
        raise HTTPException(status_code=401)

    # Check password
    valid = bcrypt.checkpw(
        form_data.password.encode(),
        fake_user["hashed_password"]
    )

    if not valid:
        raise HTTPException(status_code=401)

    # Create JWT token
    token = jwt.encode(
        {"sub": fake_user["username"]},
        SECRET_KEY,
        algorithm="HS256"
    )

    return {"access_token": token}

# -----------------------------
# Protected Route
# -----------------------------

@app.get("/profile")
def profile(token: str = Depends(oauth2_scheme)):

    data = jwt.decode(
        token,
        SECRET_KEY,
        algorithms=["HS256"]
    )

    return {
        "message": "Protected route",
        "user": data["sub"]
    }
  • FastAPI’s official security tutorial demonstrates OAuth2 password flow with JWT tokens and hashed passwords.
  • Use a secret key, expiry time, and secure password hashing.
  • Role-based access can be layered on top of the current-user dependency.
Never store plain passwords. Hash them before persistence, and validate tokens on every protected request.

HTTPException and global exception handlers

from fastapi import HTTPException

@app.get("/students/{student_id}")
def get_student(student_id: int):
    student = find_student(student_id)
    if not student:
        raise HTTPException(status_code=404, detail="Student not found")

Files, background tasks, docs

Useful operational features

FastAPI handles more than JSON. It can receive files, process forms, kick off background work, and expose docs for testing.

File uploads, forms, background tasks, API docs

from fastapi import FastAPI, File, UploadFile, Form, BackgroundTasks

app = FastAPI()

# Background task function
def save_log(username: str):
    with open("log.txt", "a") as f:
        f.write(f"{username} uploaded a file\n")

# Upload route
@app.post("/upload")
async def upload_file(
    background_tasks: BackgroundTasks,
    username: str = Form(...),
    file: UploadFile = File(...)
):

    # Run task in background
    background_tasks.add_task(save_log, username)

    return {
        "username": username,
        "filename": file.filename
    }
  • Use file and form helpers for browser-style submissions.
  • BackgroundTasks is useful for quick follow-up work such as emails or audit logging.
  • Swagger UI makes it easy to test uploads and token-protected routes without writing client code first.

items = []

@app.get("/")

@app.get("/admin")

    if x_token != "secret":
